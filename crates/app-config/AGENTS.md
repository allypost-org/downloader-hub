# app-config

Config infrastructure shared by every binary. Provides the `Config::init_parsed()` / `Config::global()` pattern documented in root `AGENTS.md`.

## Layout

- `src/common/` — shared config types referenced across binaries: `DatabaseConfig`, `peer_comms/` (worker + common), `EndpointConfig`, `TaskConfig`, `RequestConfig`, `DisabledEntries`, `ProjectConfig`, `ProgramPath`.
- `src/conditional/` — opt-in config blocks per binary (`server`, `cli`, `telegram_bot`, `discord_bot`). Each binary pulls only the blocks it needs.
- `src/traits/` — `GlobalConfig` + `Dumpable` traits (impls generated by `app-macros`).
- `src/validators/` — clap validators (`directory`, `file`, `str`, `url`).
- `src/timeframe.rs` — human-readable duration parsing (`"1 day"`, etc.).
- `src/lib.rs` — re-exports + `BUILD_DATE` / `BUILD_RUSTC_VERSION` (compile-time constants).

Each binary owns its own `config::Config` struct composed of common + conditional blocks; the `GlobalConfig` derive (from `app-macros`) backs the process-wide singleton.
